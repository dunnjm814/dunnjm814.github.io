What is mySampler?

An interactive web application bringing a minimalist audio workflow into the browser environment. This web applications was developed with React.js, Redux, Javascript, and CSS with a Python, Flask, SQLAlchemy backend, tied together with PostgreSQL as the database. Tone.Js and the web audio api were used for audio playback, audio scheduling, audio signal generation, and routing/manipulation. AWS S3 cloud storage was put to good use here to house all the users audio clips and send back a URL to store in the database.

Scheduling samples

Each of the eight input fields accepts an audio file (.wav, .mp3). The audio file makes its way through the the API routes as a put request, where it finds the appropriate sample, slot on the sampler instance, stores the file into aws, brings the url back down to store in the database. On the way back to the UI, the redux store updates and informs the user of a sample upload by conditionally rendering the file input style red. In the audio players the url is passed to web audio api through a buffer, an audio node is created in browser, and dropped into the Tone.Player() instance. This is mapped to key press events for testing and playing along. The sequencer is a event scheduler when to play the audio, depending on what row is enabled, at a specificly scheduled column. To make the sequence more musically interesting I also created a Tone.Polysynth() instance to play over the samples, and from a drop down menu a user may select whatever musical scale they would like the synth to play.
